# Figma 填充数据审查与替换（figma-mockdata-reviewer）

用于审查 Figma 设计稿中的填充文本数据是否真实、自洽、符合业务场景，并在用户确认后自动完成批量替换。这个 skill 关注的不只是文本表面正确，还包括演示可信度和业务逻辑一致性。

## 核心能力

- 读取 Figma 页面中的文本节点、截图和页面结构，理解界面场景。
- 审查时间、日期、ID、金额、状态、用户名等 mock data 的合理性与一致性。
- 识别逻辑错误、不够真实和可进一步优化的数据问题。
- 输出逐条替换建议，并保留原值、建议值和修改理由。
- 在用户确认后，对目标文本节点执行批量替换而不改动样式。

## 适用场景

- 用户希望检查设计稿中的示例数据是否真实可信。
- 需要统一订单号、金额、时间、状态、用户名等界面中的 mock data。
- 设计稿要用于评审、汇报或 demo，希望数据更贴近真实业务。
- 用户已经有明确的 Figma 文件，并希望在确认后直接批量替换文本。

## 不适用场景

- 用户想做的是设计结构或交互层面的方案评审，此时更适合 `design-selfcheck`。
- 用户需要先理解 PRD 和页面范围，此时更适合 `prd-design-brief`。
- 用户没有可访问的 Figma 文件，也没有可供分析的截图或页面范围说明。

## 输入与前置条件

- 需要用户提供 Figma 文件链接，或确保目标文件已在当前上下文可访问。
- 调用 `use_figma` 前，必须先加载 `figma-use` skill。
- 如果文件包含多个 Page，默认只分析用户指定页面或当前可见页面；全量处理前应先确认范围。
- 审查与替换依赖 Figma 相关工具，包括 `get_design_context`、`get_screenshot` 和 `use_figma`。

## 使用方式

### 推荐指令

```text
帮我检查这个 Figma 设计稿里的填充数据是否真实一致：https://www.figma.com/design/...
```

```text
请审查一下这几个页面的 mock data，重点看时间、金额、状态和订单号。
```

```text
把这个页面里明显不合理的假数据列出来，确认后帮我批量替换。
```

### 建议同时提供的信息

- 指定分析范围：某个页面、某个 Frame，或当前 Page。
- 指定关注字段：如时间、金额、用户信息、订单数据等。
- 指定业务场景：如订单详情、用户管理、数据看板、聊天列表等。

## 工作流程

1. 读取设计截图和页面结构，提取目标范围内的文本节点。
2. 结合页面上下文，判断当前界面的业务场景和数据语义。
3. 按时间日期、ID 编号、数值、状态联动、用户信息和业务常识逐项审查。
4. 输出有序的改进建议列表，并标注问题级别与替换理由。
5. 待用户确认后，批量替换对应文本节点，并返回修改摘要。

## 输出结果

- `改进建议列表`：逐条列出模块名、字段名、当前值、建议值和修改理由。
- `严重性分级`：
  - `逻辑错误`：会造成明显误解，如时间倒序、汇总不一致。
  - `不够真实`：不影响理解，但不符合常见业务认知。
  - `优化建议`：更自然、更美观或更符合行业习惯。
- `确认选项`：支持全部执行、仅执行逻辑错误项、手动指定编号或仅参考不执行。
- `替换结果摘要`：列出已修改条目和替换数量，便于回查。

## 注意事项

- 批量替换前必须取得用户确认，不能直接改写设计稿。
- 替换只修改字符内容，不应改变字体、颜色、字号等样式。
- 如果文本节点使用混合字体，执行替换前需要先加载对应字体。
- 建议在操作前确认处理范围，避免误改无关页面。
